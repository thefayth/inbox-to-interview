# Public / Private Boundary

## Public

- Project name, purpose, and status.
- Public-safe workflow diagrams.
- Brand assets selected for project presentation.
- WordPress page draft.
- Ownership and policy documents.

## Private

- Source code.
- Prompts and agent instructions.
- Mailbox content and recruiter threads.
- Resume sources, private facts, personal records, and application history.
- Database, credentials, tokens, logs, infrastructure, and deployment details.

## Rule

Public materials may explain what the project does. They must not expose how the private engine operates or any protected data.
